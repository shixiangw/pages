1. 变更概览

PR #1253 — ECP-Neuron 代理超时、连接池、JWT 泄漏修复

修复了 ECP-Neuron 代理服务在生产环境中发现的四个关键缺陷：代理超时、连接池耗尽、JWT Token 内存泄漏和并发数据竞争。同时引入了 TTL 泛型缓存机制以缓解 DB 查询压力，并完善了优雅关闭流程。

此 PR 是 P0 级别修复，变更范围覆盖 CI/CD、数据访问层、代理层、认证层、JWT 层及服务启动/关闭生命周期，涉及模块广、影响面大。

PR #1254 — JWT 私钥加载重试 & 离线判定窗口扩大

修复两个问题：(1) JWT 私钥加载使用 sync.Once 导致加载失败后永久缓存错误，启动后再也无法恢复，改为可重试加载；(2) 边缘服务离线判定窗口从 2×healthInterval 扩大到 100×，避免网络抖动导致误判离线。代码评审中同步重构了 JWT 加载实现（Mutex 替代 RWMutex、常量化魔数、统一转换模式）。

此 PR 为 P1 优化，离线窗口扩大仅影响统计归类和 UI 兜底判定，不改变核心写入路径（K8s/直连/代理服务仍直接写 Off）。

4. 风险评估

高风险 — UpdateOrgProjectAndAgentID 签名变更 (PR1253)

logic/edgeservice/edge.go 中的 UpdateOrgProjectAndAgentID 返回值从 error 变为 ([]string, error)。本次变更中调用方已适配，但需排查其他分支是否有未修改的调用点。

高风险 — serviceId2JwtMap 变量名变更 (PR1253)

从 ServiceId2JwtMap（导出）改为 serviceId2JwtMap（非导出），类型从 map[string]string 改为 map[string]tokenEntry。外部引用会导致编译失败。

中风险 — 离线窗口从 2×→100× 的业务影响 (PR1254)

离线判定窗口扩大 50 倍（如 healthInterval=30s 时从 60s 变为 3000s）。虽然核心路径（K8s/直连/代理）不受影响，但统计归类和 UI 兜底判定会出现长时间不刷新离线状态的情况，需确认业务可接受。

中风险 — 缓存一致性问题 (PR1253)

Cache-Aside 模式依赖所有写路径手动 Invalidate。本次在关键路径覆盖了 18 个 Invalidate 点，但未来新增写操作可能遗漏。

中风险 — 缓存容量与 TTL (PR1253)

EdgeServiceCache (cap=100, ttl=5min) 和 UserCache (cap=10, ttl=5min) 容量有限。当边缘服务数超过 100 时可能缓存击穿。

低风险 — 连接池参数调整 (PR1253)

MaxOpenConns 25→50、MaxConnsPerHost 10→100 是常规扩容。需确认 DB 和下游服务容量支持。

低风险 — JWT 私钥加载重构 (PR1254)

从 RWMutex 双检锁简化为 Mutex 双检，函数从 VerifyPrivateKey() 重命名为 GetPrivateKey()。行为等价、代码更简洁。所有引用已更新（含 main.go），编译通过。
