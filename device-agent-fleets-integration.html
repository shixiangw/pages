device-agent-bridge (Go)

位于 fleets 内部
  • 通过 EMQX REST API 注入/删除 3 条规则
  • 维护 da_bridge_mappings 映射表
  • 规则健康监控
  • 复用 fleets 的 EMQX_API_KEY/SECRET
  • 不订阅 MQTT
  • 不做协议转换 (规则 SQL 完成)
  • 不做设备发现

fleets-bridge (TypeScript)

位于 device-agent 内
  • 设备注册的唯一入口
  • 监听 DeviceService 事件 → 调用 Fleets API
  • 创建 ThingType + Thing (含 da-product-id / da-device-id tags)
  • 状态/影子同步
  • 本地 SQLite 映射 + 同步队列
  • 配置: Fleets API Key/Secret

设备 PUBLISH (设备 → Gateway)

Telemetry

v1/{productId}/{deviceId}/telemetry PUB
{
  "type": "status",          // "state"|"status"|"event"
  "data": {
    "temp_a": 23.5,
    "status": "online"        // type=status 时含此字段
  },
  "ts": 1710000000000         // 可选, Unix ms
}
type=status 时 data 可选包含 state 子对象作为完整快照

Event

v1/{productId}/{deviceId}/event PUB
{
  "type": "event",
  "data": {
    "event": "overheat",      // ★ 事件标识 (必须)
    "pressure_percent": 91,
    "free_percent": 6.4
  },
  "ts": 1710000000000
}
data.event 必须匹配产品模型定义的事件名

Command Response

device-agent/{productId}/device/{deviceId}/responses PUB
{
  "code": 0,                  // 0=成功, 非0=失败
  "msg": "ok",
  "requestId": "req-abc-123", // ★ 与命令 requestId 匹配
  "data": { "interval": 30 },
  "ts": 1710000000000
}

NTP Request

device-agent/{productId}/device/{deviceId}/ntp/request PUB
{
  "deviceSendTime": 1710000000000
}

Fleets 原生设备 MQTT 协议规范 topics.go setup-emqx-rule.sh

设备 Topic (设备 PUBLISH 到 fleets 主题域)

Shadow Update

$emqx/things/{thingName}/shadow/update PUB
{
  "state": {
    "reported": {
      "temperature": 23.5,
      "humidity": 60.1
    }
  }
}
★ Fleets 规则: FROM "$emqx/things/+/shadow/update" WHERE is_not_null_var(payload.state.reported)
动作: POST /api/v1/thing-datas/shadow/reported/pg + GreptimeDB INSERT

Event

$emqx/things/{thingName}/events/{eventType} PUB
{
  "eventType": "overheat",
  "severity": "info",          // info / warn / error
  "data": {
    "temperature": 85.3
  }
}
★ Fleets 规则: FROM "$emqx/things/+/events/+" WHERE is_not_null_var(payload.eventType)
动作: GreptimeDB INSERT (fleets_events 表)

Command Response

$emqx/commands/things/{thingName}/executions/{executionId}/response PUB
{
  "status": "succeeded",       // succeeded / failed
  "result": { "interval": 30 }
}
★ 注意命名空间: $emqx/commands/things/{name}/... (不是 $emqx/things/...)
Fleets 规则: FROM "$emqx/commands/things/+/executions/+/response"

Shadow Delta (SUB)

$emqx/things/{thingName}/shadow/update/delta SUB
{
  "version": 5,
  "state": { "desired": { "interval": 30 } }
}
设备订阅此主题接收 fleets 下发的 desired 状态

Jobs 相关 Topic

$emqx/things/{name}/jobs/get $emqx/things/{name}/jobs/start-next $emqx/things/{name}/jobs/{jobId}/get $emqx/things/{name}/jobs/{jobId}/update

三套 EMQX Rule SQL (device-agent-bridge Plugin 模板)
#1 Telemetry: v1/+/+/telemetry → $emqx/things/da:*/shadow/update ▾ 
SELECT
  concat('$emqx/things/da:',
    (regex_match(topic, '^v1/([^/]+)/([^/]+)/telemetry$'))[1], '/',
    (regex_match(topic, '^v1/([^/]+)/([^/]+)/telemetry$'))[2],
    '/shadow/update'
  ) as target_topic,

  json_encode(map(
    'state', json_encode(map(
      'reported', coalesce(
        json_decode(payload)->'data'->'state',
        json_decode(payload)->'data'
      )
    ))
  )) as target_payload

FROM "v1/+/+/telemetry"
动作: 
{"function":"republish","args":{"topic":"${target_topic}","payload":"${target_payload}","qos":1,"retain":false}}
#2 Event: v1/+/+/event → $emqx/things/da:*/events/{eventName} ▾ 
SELECT
  concat('$emqx/things/da:',
    (regex_match(topic, '^v1/([^/]+)/([^/]+)/event$'))[1], '/',
    (regex_match(topic, '^v1/([^/]+)/([^/]+)/event$'))[2],
    '/events/',
    json_decode(payload)->'data'->>'event'
  ) as target_topic,

  json_encode(map(
    'eventType', json_decode(payload)->'data'->>'event',
    'severity',  'info',
    'data',      json_decode(payload)->'data'
  )) as target_payload

FROM "v1/+/+/event"
#3 Command Response: device-agent/+/device/+/responses → $emqx/commands/things/da:*/executions/*/response ▾ 
SELECT
  concat('$emqx/commands/things/da:',
    (regex_match(topic, '^device-agent/([^/]+)/device/([^/]+)/responses$'))[1], '/',
    (regex_match(topic, '^device-agent/([^/]+)/device/([^/]+)/responses$'))[2],
    '/executions/',
    json_decode(payload)->>'requestId',
    '/response'
  ) as target_topic,

  json_encode(map(
    'status', iif(json_decode(payload)->>'code' = '0', 'succeeded', 'failed'),
    'result', coalesce(json_decode(payload)->'data', json_decode('{}'))
  )) as target_payload

FROM "device-agent/+/device/+/responses"

核心接口
type BridgePlugin interface {
    Name() string
    Init(BridgeDependencies) error
    Start(ctx context.Context) error
    Stop(ctx context.Context) error
    IsEnabled() bool
    Status(ctx) (*PluginStatus, error)
}
启动 Start() 流程
  1. 运行 DB 迁移 (da_bridge_mappings 表)
  2. 检查 EMQX API 连通性
  3. 按规则 ID 前缀查重 (da-bridge-*)
  4. 创建 3 条规则 (若不存在)
  5. 启动健康检查协程
不负责
  • 订阅 MQTT topic
  • 设备发现/自动注册
  • 协议转换 (规则 SQL 完成)
  • 命令下发 MQTT (复用 emqx.Publisher)

注册流程 (唯一入口)
  1. 监听 DeviceService 的 device:created 事件
  2. 检查本地 SQLite 映射 → 存在则跳过
  3. 确保 Fleets ThingType 存在 (POST /api/v1/thing-types)
  4. 创建 Thing (POST /api/v1/things)
  5. 响应包含 da-product-id, da-device-id tags
  6. Fleets ThingService 自动写入 da_bridge_mappings
  7. DA Plugin 保存本地映射 (SQLite)
  8. 失败入同步队列 (指数退避重试)
